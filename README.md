# Laminated-glass- — Data-Driven Predictive Framework for Impact Threshold

A comprehensive machine-learning–based predictive framework for determining the **rebound–penetration threshold kinetic energy** of laminated glass panels subjected to lumbar (soft-body) impact, developed using data generated from finite element simulations in **LS-DYNA**.

## Problem Overview

Laminated glass panels are widely used in architectural and automotive glazing.  When subjected to impact loads, a panel either **rebounds** (absorbs and returns the impactor) or undergoes **penetration** (the glass is perforated).  Identifying the exact transition — the *threshold kinetic energy* — for a given panel configuration and impactor mass is critical for structural safety design.

This repository provides:
- A representative **LS-DYNA–style dataset** with two laminated-glass configurations.
- A **MATLAB script** that trains classification models, locates the SVM decision boundary, and reports the threshold KE for every (configuration, impactor-mass) combination.

## Dataset Description

The dataset (`Data_LaminatedGlass.csv`) contains simulation results for two laminated glass configurations:

| Config | Ply | Lay-up | Total thickness |
|--------|-----|--------|-----------------|
| 1 | 3-ply | 5 mm Glass / 1.52 mm PVB / 5 mm Glass | 11.52 mm |
| 2 | 5-ply | 4 mm Glass / 0.76 mm PVB / 4 mm Glass / 0.76 mm PVB / 4 mm Glass | 13.52 mm |

### Columns

| Column | Unit | Description |
|--------|------|-------------|
| `Config` | — | Configuration ID (1 = 3-ply, 2 = 5-ply) |
| `t_g1_mm` | mm | Thickness of first glass layer |
| `t_il1_mm` | mm | Thickness of first interlayer (PVB/EVA) |
| `t_g2_mm` | mm | Thickness of second glass layer |
| `t_il2_mm` | mm | Thickness of second interlayer (0 for 3-ply) |
| `t_g3_mm` | mm | Thickness of third glass layer (0 for 3-ply) |
| `Total_Glass_mm` | mm | Total glass thickness |
| `Total_Interlayer_mm` | mm | Total interlayer thickness |
| `Mass_kg` | kg | Impactor mass (1, 2, 3, or 4 kg) |
| `Velocity_ms` | m/s | Impact velocity |
| `KE_J` | J | Initial kinetic energy ( ½mv² ) |
| `Status` | — | **0 = Rebound**, **1 = Penetration** |
| `Central_Deflection_mm` | mm | Maximum central panel deflection |
| `Residual_Velocity_ms` | m/s | Impactor residual velocity (0 for rebound) |

## Methodology

### Feature Engineering
In addition to the raw geometric and impact parameters, the following composite features are computed:
- **Total panel thickness** = Total_Glass + Total_Interlayer
- **Glass-to-interlayer ratio** = Total_Glass / Total_Interlayer
- **Number of plies** (3 or 5)
- **Specific kinetic energy** = KE / Mass

### Classification Models Trained
| Model | Description |
|-------|-------------|
| **SVM (RBF kernel)** | Primary model; uses Gaussian radial basis function kernel |
| Decision Tree | Fine tree with Gini impurity splitting criterion |
| Logistic Regression | Ridge-regularised linear classifier |

All models are evaluated via **5-fold stratified cross-validation**.

### Threshold KE Computation (SVM Decision Boundary)
For each `(configuration, impactor mass)` pair:
1. A fine KE sweep (1 J → 80 J, step 0.01 J) is generated.
2. Corresponding velocity and all features are constructed.
3. The trained SVM predicts a raw decision score for each point.
4. The KE at which the score crosses zero (rebound → penetration) is found by **linear interpolation** — this is the threshold KE.

## How to Run

### Prerequisites
- MATLAB R2021a or later
- Statistics and Machine Learning Toolbox

### Steps
1. Clone or download this repository.
2. Open MATLAB and navigate to the repository folder:
   ```matlab
   cd('path/to/Laminated-glass-')
   ```
3. Run the main script:
   ```matlab
   LaminatedGlass_ThresholdPrediction
   ```
4. The script will:
   - Print a summary table to the command window
   - Generate **8 publication-quality figures**
   - Save `Threshold_KE_Results.csv` to the current folder

## Using Your Own Data

### From an Excel file
Replace the `readtable` call at the top of the script:
```matlab
T = readtable('YourFile.xlsx');
```

### Column name mapping
If your Excel file uses different column names, update the column-name variables in **Section 1** of the script (lines marked `col*`):
```matlab
colConfig   = 'Config';        % configuration ID column
colTg1      = 't_g1_mm';       % first glass thickness
colTil1     = 't_il1_mm';      % first interlayer thickness
% ... etc.
```

### Adding more configurations or masses
The script is fully data-driven — it automatically detects the unique configuration IDs and impactor masses present in the dataset and processes each combination without any hard-coded values.
